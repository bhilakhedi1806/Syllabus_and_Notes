Python Namespace
  • Name????
  • Name (also called identifier) is simply a name given to objects. Everything in Python is an object.
  • Name is a way to access the existing object.
  • For example,
  • a=2, 2 is an object stored in memory and a is the name associate it.
  • We can get the address (in RAM) of some object through the built-in function id().
  • Example:
a = 2
print('id(2) =', id(2))
print('id(a) =', id(a))
  • Output
id(2) = 9302208
id(a) = 9302208
Here, both refer to the same object 2, so they have the same id().
  • Example:
a = 2
print('id(a) =', id(a))
a = a+1
print('id(a) =', id(a))
print('id(3) =', id(3))
b = 2
print('id(b) =', id(b))
print('id(2) =', id(2))
  • Output
id(a) = 9302208
id(a) = 9302240
id(3) = 9302240
id(b) = 9302208
id(2) = 9302208
  • Initially, an object 2 is created and the name a is associated with it, when we do a=a+1, a new object 3 is created and now a is associated with this object.
  • Note that id(a) and id(3) have the same values.
  • Furthermore, when b=2 is executed, the new name b gets associated with the previous object 2.
  • This is efficient as Python does not have to create a new duplicate object.
  • This dynamic nature of name binding makes Python powerful

Namespace

  • A namespace is a collection of names.
  • A namespace is a system to have a unique name for each and every object in Python.
  • An object might be a variable or a method or a class.
  • Python itself maintains a namespace in the form of a Python dictionary.
  • Real-time example
  • The role of a namespace is like a surname.
  • One might not find a single “Chouhan” in the class there might be multiple “Chouhan” but when you particularly ask for “Sachin Chouhan” or “Deepak Chouhan” (with a surname), there will be only one.
  • Python interpreter understands what exact method or variable one is trying to point to in the code, depending upon the namespace.
  • So, the division of the word itself gives little more information. Its Name(which means name, an unique identifier) + Space(which talks something related to scope).
  • Here, a name might be of any Python method or variable and space depends upon the location from where is trying to access a variable or a method.
